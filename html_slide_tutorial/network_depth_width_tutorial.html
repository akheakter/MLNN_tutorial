Quick MLP Refresher

Multi-Layer Perceptron Fundamentals

  • Structure: Input → Hidden Layer(s) → Output
  • Parameters: Weights + Biases between all layers
  • Parameter Count: (Layer₁ × Layer₂) + (Layer₂ × Layer₃) + ... + biases
  • Key Insight: Total parameters = computational budget
# Parameter calculation example
def count_parameters(layers):
    total = 0
    for i in range(len(layers)-1):
        total += layers[i] * layers[i+1] + layers[i+1]
    return total

Why These Differences Happen

The Science Behind the Results

Deep Networks

  • ✓ Build hierarchical representations
  • ✓ Better for complex, non-linear patterns
  • ✗ More prone to vanishing gradients
  • ✗ Slower initial training

Shallow Networks

  • ✓ Learn many simple patterns simultaneously
  • ✓ Faster training on simple problems
  • ✓ More stable gradients
  • ✗ Limited representational capacity

Universal Approximation Theorem

Any continuous function can be approximated by a single hidden layer network with enough neurons, but depth can make this more efficient

Practical Decision Guidelines

When to Choose Each Architecture

Go Deep & Narrow


✓ Hierarchical problem structure

✓ Complex non-linear relationships

✓ Time for longer training

✓ Want best possible accuracy

Go Shallow & Wide


✓ Relatively simple problem

✓ Need fast training/inference

✓ Linear relationships dominate

✓ Want quick results

Go Balanced


✓ Unsure about problem complexity

✓ Want robustness across types

✓ Need a safe default choice

✓ Moderate computational budget
